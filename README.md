**Тема:** Разработка программного модуля для учета заявок на ремонт климатического оборудования (Вариант 5)

## Описание проекта
Разработка веб-приложения для автоматизации учёта заявок на ремонт климатического оборудования (кондиционеры, вентиляция, увлажнители и т.д.). Система позволяет регистрировать заявки, назначать ответственных, отслеживать статусы, вести статистику, а также добавлять роли и дополнительные функции согласно техническому заданию демонстрационного экзамена.

## Основные функции модуля
- Добавление, редактирование и просмотр заявок на ремонт.
- Регистрация заявок оператором.
- Обработка и назначение исполнителя (специалиста).
- Исполнение заявки с возможностью добавления комментариев и информации о запчастях.
- Отчётность: количество выполненных заявок, среднее время ремонта, статистика по типам неисправностей.
- Мониторинг и анализ эффективности работы сервисной службы.
- **Дополнительно (по Прил_3):**
  - Роль «Менеджер по качеству» с правами: привлечение специалистов, продление срока заявки.
  - Генерация QR-кода со ссылкой на форму обратной связи для оценки качества обслуживания.

## Стек технологий

### Backend
- C#
- ASP.NET Core 8 (Web API)
- Entity Framework Core (ORM)
- PostgreSQL 17
- Инструменты: DBeaver, pgAdmin 4

### Frontend
- HTML5, CSS3
- JavaScript (нативный, без фреймворков)
- Адаптивный интерфейс с модальными окнами

## Этапы разработки
1. Проектирование базы данных:
   - Анализ предметной области и технического задания.
   - Составление ER-диаграммы в 3НФ (сущности: пользователи, заявки, комментарии, роли, типы оборудования и др.).
   - Создание таблиц в PostgreSQL, установка связей и ограничений.

2. Импорт начальных данных:
   - Загрузка данных из предоставленных файлов (inputDataRequests.txt, inputDataComments.txt, inputDataUsers.txt) в созданные таблицы.

3. Создание ASP.NET Core Web API проекта:
   - Настройка подключения к БД через Entity Framework.
   - Создание моделей (Entity) и DTO (Data Transfer Objects).
   - Реализация контекста БД (ApplicationDbContext).

4. Разработка контроллеров (CRUD):
   - Контроллеры для работы с заявками, пользователями, комментариями.
   - Обработка бизнес-логики (расчёт статистики, поиск, фильтрация).

5. Реализация авторизации и аутентификации:
   - Хэширование паролей (BCrypt).
   - JWT-токены для доступа к API.
   - Разграничение прав доступа по ролям (оператор, специалист, заказчик, менеджер по качеству).

6. Разработка клиентской части (HTML + CSS + JS):
   - Вёрстка страниц согласно макету/требованиям (список заявок, форма создания/редактирования, профиль).
   - Написание JS-скриптов для взаимодействия с API (fetch-запросы).
   - Создание модальных окон для удобства пользователя.

7. Реализация дополнительного функционала (Приложение 3):
   - Добавление роли «Менеджер по качеству» с соответствующими правами.
   - Генерация QR-кода (например, с помощью библиотеки QRCoder на бэкенде или через JS-библиотеку) со ссылкой на Google Forms.

8. Тестирование и отладка:
   - Модульное тестирование ключевых функций.
   - Интеграционное тестирование API (Postman или Swagger).
   - Функциональное тестирование интерфейса.

9. Подготовка отчёта по практике:
   - Сбор скриншотов, описания алгоритмов, блок-схем.
   - Оформление пояснительной записки (часть 1 – выполнение заданий ДЭ, часть 2 – описание практической части ВКР).
